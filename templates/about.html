About

DocuQuiz is a web-based tool that converts PDF documents into multiple-choice questions using a generative pretrained transformer. It is designed for students, educators, and self-learners who want to transform study material into an interactive quiz format for systematic revision and self-assessment.

How to use

  1. Navigate to the Upload PDF page and select a PDF file from your local filesystem.
  2. Specify the number of questions to generate (range: 1–30).
  3. Choose a difficulty level that determines the cognitive depth of the generated items — Easy (recall-oriented), Medium (comprehension-oriented), or Hard (application and analysis-oriented).
  4. Click Generate MCQs to submit the document for processing and generate a quiz, or click Listen to PDF to navigate directly to the text-to-speech reader for the extracted document.
  5. Review each question interactively: click on any answer option to receive immediate feedback. Correct selections are highlighted in green, incorrect ones in red, and the correct answer is revealed automatically.
  6. Past quizzes are persisted in the browser's local storage and can be revisited from the History page at any time without requiring a new generation cycle.

How it works

  • Text extraction — The uploaded PDF is processed through a standard text extraction pipeline that parses the document structure and extracts raw textual content from embedded text layers.
  • Contextual input preparation — The extracted corpus is cleaned, segmented, and formatted into a structured prompt that preserves semantic relationships across sections, paragraphs, and sentences.
  • Question generation via generative pretrained transformer — The prepared text is passed as contextual input to a generative pretrained transformer, which identifies salient concepts, relationships, and domain-specific terminologies within the corpus.
  • Distractor synthesis — For each identified concept, the model generates a question stem along with four answer options: one factually correct answer grounded in the source text, and three plausible distractors designed to test discriminative comprehension.
  • Difficulty modulation — The user-selected hardness parameter (Easy, Medium, or Hard) controls the specificity and cognitive depth the model is instructed to target. Easy maps to recall-level questions, Medium to comprehension-level, and Hard to application and analysis-level reasoning.
  • Response formatting — The model output is parsed into a structured JSON payload containing the question stem, an ordered dictionary of options (A–D), and the designated correct answer. This payload is rendered into an interactive quiz interface on the client side.

Listen to PDF

  • Text-to-speech reader — The Listen tab provides a dedicated page for hearing the extracted PDF content read aloud. The text is displayed alongside a player panel with Play/Pause, Stop, and speed controls (0.5×–2.0×).
  • Two entry points — You can reach the Listen page by clicking Listen to PDF on the upload form, or by navigating directly to the Listen tab in the navigation bar.
  • Client-side speech — All speech is generated locally using the browser's built-in Web Speech API. No audio data is sent to any server. The extracted text is stored in the browser's local storage for playback.
  • Playback controls — Use the Play/Pause toggle to start and pause reading. The Stop button resets playback to the beginning. The Speed slider lets you slow down or speed up the narration to suit your preference.

Tips for best results

  • Text-based PDFs yield optimal results. Scanned documents or image-only PDFs rely on embedded text layers; since no optical character recognition (OCR) pipeline is integrated, such files may fail to produce usable output.
  • The Hard difficulty setting is most effective when the source material contains dense, technical, or multi-layered content that supports analysis-level questioning.
  • Files must not exceed 16 MB due to server-imposed upload constraints.
  • The History tab provides persistent access to all previously generated quizzes. Entries can be viewed in full or deleted individually. All data resides exclusively in the client's local storage and does not transit to any external server.
  • The Listen tab requires a browser that supports the Web Speech API (Chrome, Edge, Safari, Firefox). Speech is generated entirely on your device.

Privacy

DocuQuiz processes uploaded PDFs entirely in server memory. Once text extraction is complete, the uploaded file is deleted from the server filesystem before the response is returned. No PDF content, extracted text, or generated quiz data is retained, logged, or transmitted to any third party. Quiz history is stored solely in the browser's localStorage API and remains under the user's exclusive control.
